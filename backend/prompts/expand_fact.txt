You expand one analytical point (a counterargument or further support) into a small argument subgraph
that will be merged into an existing map. The user clicked "Add" on this point to grow the graph.

Parent node id (must appear as target in edges from new nodes where applicable): {parent_node_id}
Parent node type: {parent_type}
Parent label: {parent_label}
Parent detail: {parent_detail}

Kind of fact: {fact_kind}
Fact text to expand: {fact_text}

Original passage for context:
{original_text}

Return ONLY valid JSON. No markdown, no code fences.

All node types allowed in this API are: thesis, subclaim, evidence, axiom, counterclaim.

Every new node MUST include ALL of the following (never omit keys; never use empty arrays []):
- id, type, label, detail, strength (one of true|strong|weak|fallacious|false)
- counterarguments: exactly 2–3 short strings
- further_supports: exactly 1–3 short strings
- strength_reasoning: one sentence explaining the strength label

If the passage is thin, infer plausible counterarguments and strengths from the node label, detail, and fact text.

---

### When fact_kind is "counterargument" (MANDATORY SHAPE)

Produce **2–4 NEW nodes** total:

1. **Exactly one** node with **type "counterclaim"**. It is the objection to the parent. It MUST be the **only** new node that links to the parent.

2. **At least one** additional node whose **type is only one of**: **evidence**, **subclaim**, or **axiom**. These support or spell out the counterclaim.

3. **Edges (child → parent in this graph’s convention)**:
   - One edge: **source** = the counterclaim’s id, **target** = "{parent_node_id}", **relation** = "contradicts".
   - For each evidence/subclaim/axiom node: **source** = that node’s id, **target** = the **counterclaim’s id**, **relation** = "supports".

4. Do **not** connect evidence/subclaim/axiom directly to "{parent_node_id}". Do **not** use thesis for this expansion.

5. Labels: keep them short (max 8 words).

### When fact_kind is "further_support"

Create **1–3 NEW nodes** only. Types: thesis, subclaim, evidence, axiom, counterclaim (pick what fits).

Edges: point FROM each new child TO "{parent_node_id}" (child → parent). Use **relation** "supports" (or "contradicts" only when the new material directly attacks the parent).
